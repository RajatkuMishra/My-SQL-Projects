# Technical Support Ticketing System - SQL Project

This repository contains a MySQL-based Technical Support Ticketing System designed to effectively manage and track customer support cases. It includes scripts for creating the database schema, sample data inserts, and useful SQL queries for managing tickets, agents, customers, and generating reports.

## Features
- **Database Schema**: Includes tables for Customers, Support Agents, Tickets, Categories, Statuses, and Ticket Updates.
- **Sample Data**: Provides realistic data to simulate real-world support cases and workflows.
- **SQL Queries**: Includes queries to track open tickets, understand agent workload, monitor ticket statuses, and evaluate performance.
- **Reports**: Tools for analyzing ticket trends, agent performance, and customer activity.

## Getting Started
1. Clone this repository.
2. Use MySQL to execute the `support_ticketing_system.sql` script to create the necessary tables and insert sample data.
3. Utilize the provided SQL queries to explore and manage ticket data.
4. Feel free to extend the schema and queries as needed for your specific use case.

## Requirements
- MySQL Server 5.7 or later
- MySQL Workbench or any SQL client to run the scripts

## Usage
Follow these steps to run the scripts in order:
1. Create the database and tables.
2. Insert sample data.
3. Execute the sample queries provided in the `queries.sql` file or the README.

You are welcome to customize and expand the system with additional features such as ticket priorities, SLA tracking, or user authentication.
